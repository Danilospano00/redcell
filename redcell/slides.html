01 · the problem

Who runs an AI agent in production?
Who has pentested it?

Everyone wires chatbots to databases, APIs and tools. Every connection opens a new attack surface. Almost no one tests it.

  • Direct prompt injection: steer the agent with hostile input
  • Jailbreak & system-prompt theft: extract internal rules and secrets
  • Data exfiltration via tool abuse: read what you shouldn't

It's the OWASP Top 10 for LLM. New surface, old attackers.

06 · on a third-party agent

How the patch reaches production

In the demo the victim's prompt is ours: the policy is concatenated in RAM. On a real agent the prompt belongs to the customer. RedCell writes what to change, it doesn't sneak in. Three ways, from simplest to most integrated.

01 · TODAY Patch as a diff

gpt-oss generates the policy. RedCell delivers it as a ready diff: a block to add to the system prompt plus authz checks in the tools. The team (or CI) pastes it and deploys. RedCell re-attacks the live version to verify.

02 · INTEGRATED Config hook

The customer exposes a management API. RedCell writes to it directly, Dependabot style: it opens a PR on your repo because you granted access. Programmatic append, but on their system, with their permission.

03 · DEPLOYMENT Guardrail as a proxy

RedCell doesn't touch the prompt: it sits in front of the agent. Input guardrail + output DLP (already code) run in the proxy, intercepting request and response. The defense is external: no append to the prompt.
